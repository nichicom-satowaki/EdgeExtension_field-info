# Field Info Helper - Edge拡張機能

項目定義情報の自動フィルタリング、ナビゲーション、テキストファイル出力を行うMicrosoft Edge拡張機能です。

## 機能概要

- **運用管理**ボタン（未実装）
- **予防接種**ボタン（未実装）  
- **成人保健**ボタン（未実装）
- **母子保健**ボタン：A103PregnancyDtoの自動処理とテキストファイル出力

### 母子保健ボタンの詳細機能
1. **固定DTO処理**：A103PregnancyDtoを自動でフィルタリング・検索
2. **自動ナビゲーション**：検索結果からメンバーページに自動遷移
3. **ラジオボタン選択**：「テキスト」ラジオボタンを自動選択
4. **データ抽出・出力**：form-controlデータを直接抽出してテキストファイル出力
5. **エラーハンドリング**：ページ読み込みエラーや要素検索エラーの適切な処理
6. **重複処理防止**：同一処理の重複実行を防ぐ安全機能

### 自動処理の流れ
```
母子保健ボタンクリック
    ↓
A103PregnancyDtoでフィルタ設定・検索実行
    ↓
検索結果リンクを自動クリック
    ↓
メンバーページ（/A103PregnancyDto/member）へ自動遷移
    ↓
データ読み込み完了を待機（最大60秒）
    ↓
「テキスト」ラジオボタンを自動選択
    ↓
Form-controlデータを直接抽出
    ↓
A103PregnancyDto.txtファイル自動ダウンロード
```

## インストール方法

1. Microsoft Edgeを開きます
2. アドレスバーに `edge://extensions/` と入力してEnterキーを押します
3. 右上の「開発者モード」をオンにします
4. 「展開して読み込み」ボタンをクリックします
5. この拡張機能フォルダ（EdgeExtension_field-info）を選択します
6. 拡張機能が読み込まれ、ツールバーにアイコンが表示されます

## 使用方法

### 基本的な使用方法

1. `http://localhost:8081/core/setup/field-info` にアクセスします
2. ツールバーの拡張機能アイコンをクリックしてポップアップを開きます
3. 「母子保健」ボタンをクリックします
4. 自動処理が開始され、最終的に `A103PregnancyDto.txt` がダウンロードされます

### 出力ファイルについて

- **ファイル名**: URLから自動取得（例：`A103PregnancyDto.txt`）
- **ファイル形式**: UTF-8 BOM付きタブ区切りテキスト
- **内容**: form-controlから抽出した項目定義データのみ
- **構造**: ヘッダー行 + データ行（クォートで囲まれたフィールド）

### 設定について

現在の実装では、母子保健ボタンは `A103PregnancyDto` に固定されています。
他のDTOを処理したい場合は、popup.jsの以下の行を修正してください：

```javascript
const dtoName = 'A103PregnancyDto'; // ここを変更
```

## ファイル構成

- `manifest.json` - 拡張機能の設定ファイル（Manifest V3）
- `popup.html` - ポップアップウィンドウのHTML（4つのボタン配置）
- `popup.css` - ポップアップウィンドウのスタイル（母子保健ボタンは緑色）
- `popup.js` - ポップアップウィンドウの動作ロジック（固定DTO名使用）
- `content.js` - DOM操作、自動ナビゲーション、データ抽出、ファイル出力
- `config.json` - 設定ファイル（現在は使用していない）
- `options.html/css/js` - 設定画面（現在は使用していない）
- `README.md` - この説明書

## トラブルシューティング

### 拡張機能が動作しない場合

1. 拡張機能が有効になっているか確認してください
2. 対象ページ（localhost:8081/core/setup/field-info）にアクセスしているか確認してください
3. 開発者ツール（F12）でConsoleタブのログを確認してください

### エラーメッセージが表示される場合

**「指定されたDTOを含む項目が見つかりませんでした」**が表示される場合：

1. config.jsonのDTO名が正確か確認してください
2. システムにDTOファイルが正しく配置されているか管理者に確認してください
3. 項目定義がシステムに登録されているか確認してください

### テキストファイルが出力されない場合

1. F12でConsoleタブを開き、処理ログを確認してください
2. ポップアップブロッカーが有効になっていないか確認してください
3. ダウンロード設定でファイルのダウンロードが許可されているか確認してください

### デバッグ方法

1. F12でConsoleタブを開く
2. 母子保健ボタンをクリック
3. 以下のログが表示されることを確認：
   - DTO処理開始
   - フィルタ設定完了
   - ページ遷移確認
   - データ読み込み完了
   - テキストファイル出力完了

## 開発者向け情報

### ファイル変更後の反映方法

拡張機能を変更した場合は、`edge://extensions/`ページで拡張機能の「再読み込み」ボタンをクリックして変更を反映してください。

### 主要な機能と実装

- **重複処理防止**: `isProcessingDto`フラグで同一処理の重複実行を防止
- **エラーハンドリング**: ページ全体のテキスト検索によるエラーメッセージ検出
- **自動データ抽出**: システムボタンが見つからない場合の手動テーブル抽出
- **段階的待機**: 各処理段階での適切な待機時間設定

### 処理時間・待機設定

- ページ遷移後待機: 7秒
- データ読み込み確認: 最大60秒（1秒間隔でチェック）
- テキスト選択後待機: 3秒
- エラーページ検出: 即座にチェック

### 技術的詳細

- **DOM要素検索**: Vue.js data-v属性対応の安全なCSSセレクタ使用
- **データ抽出**: form-controlクラス要素から直接値を取得
- **ファイル出力**: Blob APIを使用したクライアントサイドファイル生成
- **エラーハンドリング**: Try-catch文とコンソールログによる詳細なデバッグ情報
- **重複処理防止**: isProcessingDtoフラグによる状態管理
